# Paste-Ready Master Prompt

Implement Phase 6: Signature Proof Moments.

Use the existing docs-overhaul direction as the source of truth, but treat this phase as a targeted emphasis rebalance rather than a broad redesign.

## Goal

Keep the calm baseline and quiet-page discipline intact.
Upgrade only the flagship hiring-manager-facing proof moments so the site feels more memorable, more technically authoritative, and more visually convincing without regressing into clutter.

## Priority pages

In order:
1. homepage
2. surface repair workflow
3. eSSVI smooth handoff
4. architecture
5. local-vol / PDE validation
6. performance evidence
7. final cross-page proof-routing pass only if needed

## What must improve

- homepage gets one unmistakable signature visual
- surface repair becomes the strongest production-minded proof page
- eSSVI becomes the strongest quant-proof page
- architecture feels premium and intentional, not merely cleaned up
- local-vol / PDE reads as a real validation argument
- performance reads as an authored case study

## Core rules

- Do not make the site louder overall.
- Keep API and Quickstart quiet.
- If a treatment is flashy, it must communicate something real.
- Do not silently rewrite technical content.
- Do not touch Notes markdown content.
- Prefer source changes over generated output edits.

## 3D policy

Use 3D only where it materially improves surface/proof understanding.
Good candidates:
- homepage hero
- surface repair hero
- eSSVI hero
- maybe one supporting local-vol context figure

Do not make 3D the primary device for performance, architecture, API, or utility pages.

## Output requirements after each work package

Return results using `templates/stage-report-template.md` and include:
- changed files list
- short rationale for each change
- before/after screenshots
- wording changes, clearly separated from visual changes
- notes on risks / what still feels off
- explicit anti-regression check

## Important tone target

The site should feel like premium product-engineering documentation that demonstrates deep mathematical and technical understanding and robustness, not like a generic flashy developer portfolio.
